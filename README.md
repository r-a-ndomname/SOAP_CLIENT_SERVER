# SOAP Testing Sandbox: Spyne + Zeep

Этот проект представляет собой демонстрационный стенд для автоматизации тестирования SOAP-сервисов на Python. Он включает в себя локальный SOAP-сервер на базе **Spyne** и клиент для автотестов на базе **Zeep**.

##  Основные возможности
- **Mock-сервер**: Быстрое развертывание SOAP-сервиса с валидацией по XSD-схеме.
- **Zeep Client**: Современный клиент с поддержкой строгого типизирования запросов.
- **Logging & Inspection**: Использование `HistoryPlugin` для перехвата и анализа сырых XML (Envelope) запросов и ответов.
- **Validation**: Проверка входящих данных на стороне сервера (`validator='schema'`).

##  Технологический стек
- **Python 3.10+, но не выше 3.11.9**
- **Zeep**: Лучшая библиотека для работы с SOAP.
- **Spyne**: Библиотека для создания RPC-сервисов.
- **lxml**: Для парсинга и красивого вывода XML.
- **uv**: Быстрый менеджер пакетов.

##  Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/r-a-ndomname/SOAP_CLIENT_SERVER.git
   cd SOAP_CLIENT_SERVER
   ```

2. **Установите зависимости через `uv`:**
   ```bash
   uv pip install zeep spyne lxml
   ```

3. **Запустите сервер:**
   ```bash
   python main.py
   ```
   Сервер поднимется по адресу: `http://localhost:8000/?wsdl`

##  Что здесь тестируется 
В данном примере реализован сложный метод `Shiporder`, который позволяет проверить:
- Передачу атрибутов XML (`Orderid` как `XmlAttribute`).
- Работу с вложенными структурами (`Shipto`).
- Передачу массивов/списков объектов (`Items`).
- Валидацию обязательных полей (`min_occurs=1`, `nillable=False`).

## Структура проекта
- `server.py` — Реализация SOAP-сервиса на Spyne.
- `client_test.py` — Логика тестов, инициализация клиента и проверка ответов.
- `requirements.txt` — Список зависимостей.
